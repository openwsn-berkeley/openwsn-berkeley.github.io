void ssp_spi_init ( )

SSP-SPI bridge to hide the use of SSP instead of SPI.

Author
Xavi Vilajosana xvila.nosp@m.josa.nosp@m.na@ee.nosp@m.cs.b.nosp@m.erkel.nosp@m.ey.e.nosp@m.du, March 2012. The SPI module functions in two modes:
  • in "blocking" mode, all calls return only when the module is done. the CPU is not available while the module is busy. This is the preferred method is low-RAM system which can not run an RTOS
  • in "interrupt" mode, calls are returned after the module has started a transaction. When the transaction is done, the module uses a callback function to signal this to the caller. This frees up CPU time, allowing for other operations to happen concurrently. This is the preferred method when an RTOS is present.
